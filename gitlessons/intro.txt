GIT - распределенная система контроля версий
Это система для отслеживания и ведения истории изменения файлов, в вашем проекте.

Репозиторий - хранилище вашего кода и истории его изменений

GitHub - Веб сайт для размещения git-репозиториев в совместной разработке проектов.

Команды Git:

1. git init - она создает новый локальный гит репозиторий у вас на пк, в той директории в которой была прописана команда. (Команда вводится один раз в начале)

2. git add - команда для добавления файлов в систему отслеживания гита, после это команды изменения начинают отслеживаться гитом.

3. git commit -m '<comment>' - как только достигаем определенного момента в разработке, то тогда мы сохраняем и комментируем изменения в локальный репозиторий

4. git remote add - это команда для того чтобы связать наш локальный репозиторий с удаленным репозиторием, то есть добавляет подключение к репозиторий на гитхабе.
	git remote add <name: origin> <ссылка_на_репо:ssh>	

5. git push - команда для отправки всех ваших изменений на локальном репозитории в удаленный репозиторий
	git push origin main

6. git pull <origin> <branch> - команда при помощи которой можно стягивать изменения с удаленного репозитория

7. git clone <url> - команда для клонирования удаленного репозитория и всего его содержимого

8. git branch <name> - комнада для создания новой ветки

9. git checkout <name> - команда для переключения между ветками 

10. git revert - команда для возвращения на определнный коммит, соответсвенно для восстановления предыдущих версий кода
git revert <ссылка на коммит>

git log - можно посмотреть историю всех коммитов и посмотреть ссылки на коммиты
